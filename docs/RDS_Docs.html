Actor Framework.lvlib:Actor.lvclass:Count Actors.vi

This VI compensates for a behavior of the LabVIEW runtime engine that is normally a feature but causes problems for a system as asynchronous as the Actor Framework. The usual rule for LabVIEW is that a VI can close its front panel and keep running as long as there is an open reference to a VI. However, in a runtime engine, if the very last front panel is closed, the application is immediately terminated. This was done as an aid to users who sometimes had runaway apps and no way to abort them other than the operating system kill switch. Actors, however, operate asynchronously, and sometimes the UI will shut down long before the other actors are finished. They may still be writing to configuration files or turning off hardware. There are also times when one UI will close and another open up, and we want the actors to proceed during the interim. Thus this VI exists. It opens its own front panel in the Hidden state, which is sufficient to keep LabVIEW from aborting all VIs and quitting. It counts how many actors have been launched and closes its panel after the last actor quits. Thus for applications using the Actor Framework, as long as even one actor is still running, the automatic abort behavior of the runtime engine will not occur.

NI_PID__prctrl compat.lvlib:Get error (sp-pv).vi

This function calculates the error between the process variable and setpoint value. The error (with beta) output is used for the proportional action and the error output is used for the integral action.

NI_PID__prctrl compat.lvlib:Proportional action.vi

This function calculates the proportional action of the PID controller.

NI_PID__prctrl compat.lvlib:Integral action.vi

This function calculates the integral action of the PID controller.

A non-linear term is added to the integral action to avoid integrator windup when error values are very large. The term used is:

1 / (1 + (10 * e^2 / sp_range^2))

Therefore a large error will cause the integral action to be reduced to approximately 10%. When the error is 0, the integral action is 100%.

This VI also compensates for any change in control parameters (such as PID gains) by correcting the integrated error term to ensure smooth controller output. This method is used to prevent integrator windup when the controller output is saturated.
